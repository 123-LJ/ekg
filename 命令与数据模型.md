# EKG 命令与数据模型

## 1. 目录协议

当前仓库以 `ekg-out/` 作为默认输出根目录，主结构如下：

```text
tools/ekg/
├─ config.json
├─ ekg.json                      # 兼容镜像
├─ state.json                    # 兼容镜像
├─ experiences/                  # Experience Markdown 副本
├─ papers/                       # Paper Markdown 副本
├─ ekg-out/
│  ├─ ekg.sqlite                 # 主存储
│  ├─ ekg.json                   # 导出索引
│  ├─ state.json                 # 导出状态
│  ├─ reports/EKG_REPORT.md      # 导出报告
│  └─ panel/index.html           # 本地面板
├─ hooks/
├─ scripts/
├─ lib/
└─ tests/
```

说明：

- 正式读写优先走 SQLite。
- 根目录 `ekg.json/state.json` 仍保留为 legacy mirror。
- `experiences/` 和 `papers/` 用于阅读、审查和备份。

## 2. 命令体系

当前 CLI 可以分成 10 类。

### 2.1 基础查询类

- `stats`
- `query <keyword>`
- `trace <keyword>`
- `explain <node>`
- `path <from> <to>`

用途：

- 查询经验
- 解释图节点
- 追踪相关经验链路
- 找两点之间的图路径

### 2.2 论文与方向调研类

- `paper-query <keyword>`
- `paper-explain <paper>`
- `paper-add ...`
- `paper-import --source openalex|semanticscholar --query <keyword>`
- `survey <topic-or-keyword>`

用途：

- 查询和解释论文
- 手工新增论文
- 外部元数据导入
- 联查 papers + implementation knowledge

### 2.3 概念规范化类

- `concept-suggest <text>`
- `concept-register --canonical <key> --alias <text>`

用途：

- 给中英混合词生成 canonical suggestion
- 把 alias 注册到正式 canonical concept

### 2.4 经验写入与审查类

- `add ...`
- `review`
- `review <id> --confirm|--archive|--needs-review|--uncertain`

用途：

- 新增正式经验
- 审查已有经验质量与状态

### 2.5 Capture Candidate 类

- `capture-status [id]`
- `capture-accept <id> [--confirm]`
- `capture-dismiss <id>`

用途：

- 查看候选
- 接受进入正式图谱
- 丢弃噪声候选

### 2.6 自动提取与失效检查类

- `ingest [--source task|conversation|commit]`
- `stale-check [--baseline] [--dry-run]`

用途：

- 从任务摘要、对话、commit 生成 reviewable candidate
- 依据 anchor 漂移检查经验是否需要重新审查

### 2.7 项目上下文类

- `project-register`
- `project-list`
- `project-use`
- `project-status`
- `project-resolve`

用途：

- 维护当前工作项目
- 在多仓库环境里把文件和项目根对应起来

### 2.8 存储与导出类

- `report`
- `panel`
- `pipeline-status`
- `storage-status`
- `storage-migrate --to sqlite|json`
- `storage-rollback`

用途：

- 导出报告
- 生成或启动 panel
- 查询当前 pipeline / backend 状态
- 在 SQLite 与 JSON backend 间迁移

### 2.9 备份恢复类

- `backup-export`
- `backup-inspect`
- `backup-import`

用途：

- 导出单文件 portable backup
- 检查备份内容
- 在新环境恢复

### 2.10 调试辅助类

- `help`
- `lock-status`

## 3. 核心索引结构

当前 `index` 的核心组成是：

```json
{
  "version": "1.0.0",
  "stage": "phase-2-plus",
  "stage_label": "Phase 2+",
  "stage_summary": "SQLite, capture/review, backups, panel, graph preview",
  "generated_at": "2026-04-24T16:00:00.000Z",
  "nodes": [],
  "edges": [],
  "stats": {},
  "indexes": {}
}
```

说明：

- `nodes` 现在同时承载 Experience 和 Paper。
- `edges` 保存显式边。
- `stats` 和 `indexes` 在运行时刷新生成。

## 4. Experience 模型

Experience 是工程经验实体，当前形态如下：

```json
{
  "id": "E001",
  "kind": "Experience",
  "type": "bug-fix",
  "title": "登录重定向死循环",
  "problem": "登录后仍被守卫反复跳转",
  "solution": "排除 callback 页和已登录目标路由",
  "symptom": "",
  "cause": "",
  "fix": "",
  "scope": "",
  "root_cause": "认证守卫与重定向逻辑互相触发",
  "tags": ["auth", "redirect"],
  "techs": ["vue-router"],
  "aliases": ["signin redirect"],
  "canonical_terms": ["auth-redirect"],
  "suggested_canonical_terms": [],
  "level": "L2",
  "confidence": "CONFIRMED",
  "status": "ACTIVE",
  "source": "manual/cli",
  "project_scope": "current-project",
  "anchors": {
    "files": ["src/views/loginRedirect.vue"],
    "concepts": ["loginRedirect", "beforeEach"],
    "commits": []
  },
  "relations": ["fixes:E099"],
  "writer": {},
  "created_at": "2026-04-24T00:00:00.000Z",
  "updated_at": "2026-04-24T00:00:00.000Z",
  "experience_file": "experiences/E001-login-redirect-loop.md"
}
```

```json
  "keywords": ["signin", "callback"],
  "aliases": ["signin redirect"],
  "canonical_terms": ["auth-redirect"],
  "suggested_canonical_terms": [],
  "venue": "ICSE",
  "year": "2025",
  "url": "https://example.com/paper",

```

## 6. Edge 模型

显式边存储在 `edges` 中：

```json
{
  "from": "E001",
  "to": "P001",
  "type": "relation",
  "relation_type": "supports",
  "reason": "",
  "confidence": ""
}
```

注意：

- 图谱里还有大量运行时生成的“隐式边”。
- 例如 file/tag/topic/author/venue/canonical 等边不一定全部物化到 `edges` 表。

## 7. state 模型

`state` 用于保存运行时状态，而不仅仅是计数。

当前主要包含：

```json
{
  "version": "1.0.0",
  "stage": "phase-2-plus",
  "stage_label": "Phase 2+",
  "stage_summary": "SQLite primary storage with panel and graph preview",

    "papers": 5,

    "needs_review": 2,
    "stale": 0,

    "recent_injections": []

  },
  "pipeline": {
    "name": "ekg-build",
    "started_at": "",

  }

```

## 8. indexes 结构

当前会构建如下索引桶：

- `by_tag`
- `by_tech`
- `by_file`
- `by_status`
- `by_level`
- `by_topic`
- `by_author`
- `by_venue`

这意味着索引已经不是“只围绕经验”的模型，而是同时服务经验和论文视图。

## 9. 统计字段

当前 `stats` 至少包括：

- `experience_count`
- `paper_count`
- `active_count`
- `needs_review_count`
- `stale_count`
- `archived_count`
- `tag_count`
- `tech_count`
- `topic_count`
- `author_count`

## 10. 状态枚举

### 10.1 `confidence`

- `CONFIRMED`
- `INFERRED`
- `UNCERTAIN`

### 10.2 `status`

- `ACTIVE`
- `NEEDS_REVIEW`
- `STALE`
- `ARCHIVED`

### 10.3 `level`

- `L1`
- `L2`
- `L3`

### 10.4 `type`

Experience 常见类型：

- `bug-fix`
- `workflow`
- `decision`
- `lesson`
- `safeguard`

Paper 当前没有单独复杂 type，使用 `kind: "Paper"` 区分。

## 11. 检索配置模型

当前 `config.json` 的查询配置重点包括：

- `defaultLimit`
- `detailScoreThreshold`
- `semantic.enabled`
- `semantic.minimumScore`
- `semantic.scoreWeight`
- `semantic.tokenWeight`
- `semantic.trigramWeight`
- `semantic.fieldWeight`
- `semantic.canonicalWeight`
- `semantic.aliases`
- `multilingual.enabled`
- `multilingual.concepts`

其目标是：

- 保持规则检索的可解释性
- 增加轻量语义召回
- 支持中英混合 canonical concept

## 12. Hook 输出约定

当前 Hook 可能返回：

- 纯文本提醒
- JSON payload
- `additionalContext`
- Stop 阶段的 candidate gate 信息

典型职责：

- 编辑前提醒经验与相关论文
- 用户 prompt 提交时注入项目上下文、经验和研究线索
- 禁止直接改写受管存储
- 任务结束后生成 capture candidate

## 13. 当前模型与旧设计的主要差异

相比早期文档，当前实现已经发生了这些关键变化：

1. 主存储从“偏 JSON”切换为 SQLite first。
2. 节点从“仅 Experience”扩展为 “Experience + Paper”。
3. 索引从 tag/tech/file 扩展到 topic/author/venue。
4. 检索从纯规则扩展为规则 + 轻量语义。
5. 新增 multilingual canonical concept 体系。
6. 引入 capture candidate、project context、panel 和 backup package。

因此后续文档与实现对齐时，应以本文件为准，而不是以早期 Phase 1 草案为准。
