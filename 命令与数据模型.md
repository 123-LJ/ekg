# EKG 命令与数据模型

## 1. 目录协议

```text
tools/ekg/
├── config.json
├── ekg.json
├── state.json
├── experiences/
│   ├── E001-login-redirect-loop.md
│   └── ...
├── reports/
│   └── EKG_REPORT.md
└── hooks/
    └── pre-edit.js
```

## 2. 命令设计

### 2.1 `/ekg`

作用：

- 构建或刷新本地图谱索引
- 同步 `ekg.json`、`state.json`、`EKG_REPORT.md`

输入：

- 可选参数：`--full`、`--report-only`

输出：

- 刷新后的索引与报告
- 统计信息（经验总数、待审数、失效数）

### 2.2 `/ekg add <描述>`

作用：

- 手动新增一条经验

最小输入：

- `problem`
- `solution`

可选补充：

- `tags`
- `techs`
- `anchors`
- `level`
- `confidence`

输出：

- 新经验 ID
- 写入路径
- 摘要

### 2.3 `/ekg query <关键词>`

作用：

- 基于关键词、标签、锚点和技术对象做检索

输出：

- 候选经验列表
- 每条经验的命中原因
- 置信度、状态、最近验证时间

### 2.4 `/ekg explain <概念>`

作用：

- 汇总某个概念相关的经验

输出：

- 概念摘要
- 相关经验列表
- 频次与状态

### 2.5 `/ekg path <节点A> <节点B>`

作用：

- 查询两个节点间的关系路径

首版建议：

- 先做简单关系链返回
- 复杂 shortest path 放到 Phase 2/3

### 2.6 `/ekg review`

作用：

- 审核 `UNCERTAIN / INFERRED` 经验

操作：

- 确认
- 驳回
- 调整标签
- 提升层级

## 6. 边模型

```json
{
  "id": "R001",
  "from": "E001",
  "to": "C-auth",
  "type": "involves",
  "confidence": "CONFIRMED"
}
```

### 6.1 首版建议支持的边

| 边类型 | 含义 |
|--------|------|
| involves | 经验涉及概念或技术 |
| relates_to | 两条经验相关 |
| applies_to | 经验适用于某项目 |

## 7. 枚举定义

### 7.1 `level`

| 值 | 含义 |
|----|------|
| L1 | 实例级，绑定具体项目或实例 |
| L2 | 模式级，绑定技术栈或常见模式 |
| L3 | 原则级，抽象为跨项目可复用原则 |

### 7.2 `confidence`

| 值 | 含义 |
|----|------|
| CONFIRMED | 已确认 |
| INFERRED | 推断 |
| UNCERTAIN | 不确定，待审 |

### 7.3 `status`

| 值 | 含义 |
|----|------|
| ACTIVE | 当前有效 |
| NEEDS_REVIEW | 可能仍有效，但需要复查 |
| STALE | 过期风险高 |
| ARCHIVED | 归档 |

### 7.4 `type`

| 值 | 含义 |
|----|------|
| bug-fix | 修复类经验 |
| convention | 项目惯例 |
| decision | 设计决策 |
| lesson | 经验教训 |
| safeguard | 防回归规则 |

## 9. Hook 注入协议

### 9.1 输入

```json
{
  "session_id": "S-20260421-001",
  "target_file": "src/views/loginRedirect.vue",
  "action": "Edit"
}
```

### 9.2 输出

#### Level 0

```json
{
  "level": 0,
  "message": ""
}
```

#### Level 1

```json
{
  "level": 1,
  "message": "EKG: 当前文件命中 2 条经验（auth / redirect），可用 /ekg query redirect 查看。"
}
```

#### Level 2

```json
{
  "level": 2,
  "message": "EKG: 命中 CONFIRMED 经验 E001。问题：登录重定向死循环。建议：在全局守卫中排除已登录用户访问登录跳转页。"
}
```

## 10. 查询排序规则

首版建议排序信号：

1. 锚点命中
2. `CONFIRMED` 优先
3. `ACTIVE` 优先
4. 标签匹配数量
5. 最近验证时间

不建议首版使用：

- 复杂 embedding 分数
- 黑盒语义排序
- 无法解释的“智能推荐”

## 11. 种子经验规则

### 11.1 来源

- 当前 `skill` 库中的 L3 原则级规则
- 高复用性、低争议的工程原则

### 11.2 要求

- 必须为 L3
- 默认 `CONFIRMED`
- 不绑定某个具体项目

### 11.3 示例

```json
{
  "id": "E-SEED-001",
  "title": "全局拦截逻辑要排除自身触发路径",
  "level": "L3",

  "solution": "在进入逻辑前明确排除本次逻辑自身会重定向到的路径",
  "tags": ["routing", "guard", "redirect"],
  "techs": [],
```
