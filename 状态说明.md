# EKG 状态说明

## 当前阶段

**当前阶段标识：`phase-2-plus / phase-3-preview`**

一句话概括：

**EKG 已经具备“个人与多代理协作可实际使用”的基础质量，并且补上了经验演化治理的第一阶段与论文长文本分级存储，但距离“成熟知识中间层平台”仍然还有两步：结构化外部接入，以及更强的研究/分析工作流。**

## 已完成的核心能力

### 1. 经验层

- Experience 新增、查询、解释、路径分析已经可用
- `query / explain / trace / path / report` 已形成稳定工作流
- 支持结构化字段：`symptom / cause / fix / scope / root_cause`
- 已有 review 流程，避免未经确认的内容直接进入正式经验图谱

### 2. Capture / Review 层

- `task-complete`、`ingest` 可以生成 capture candidate
- `capture-status / capture-accept / capture-dismiss` 可用于审核
- 已补上“为什么没有 capture”的跳过原因提示，不再是静默失败
- 对长文本 summary 增加紧凑摘要压缩，减少候选噪声
- 同时保留原始长文本到 candidate origin 中，避免信息丢失

### 3. 演化治理

这一块已经不是“待设计”，而是有了第一版正式能力：

- 支持 `supersede <new> <old>` 工作流
- 旧经验不会被删除，也不会被新经验覆盖
- 旧经验会进入 `SUPERSEDED` 状态
- 查询时会降低过期版本排序，而不是直接隐藏
- `query / explain / panel` 会提示推荐当前版本和版本链上下文
- 统计与报告中已经能看到 superseded 计数

这意味着：

**经验演化现在是“非破坏式”的。旧经验仍然保留、可追溯、可解释，只是不再默认排在前面。**

### 4. 论文层

- 支持 `paper-add / paper-import / paper-query / paper-explain / survey`
- 支持 OpenAlex / Semantic Scholar 元数据导入
- 支持 canonical term、alias、suggested canonical term
- 论文节点已经进入同一知识图谱，可与经验联查

同时，长文本问题已做第一轮治理：

- 完整内容继续保留在 `abstract` 和 `paper_file`
- 检索与面板优先使用更短的 `summary`
- 新增紧凑摘要生成逻辑，减少论文长文本对 recall / review 的干扰

这属于“分级存储 + 压缩展示”的组合方案，而不是简单裁剪文本。

### 5. 存储与运行时

- SQLite 已是主存储
- JSON / Markdown 继续作为兼容镜像与可读副本
- 支持增量 SQLite 持久化
- 支持本地 panel、backup export/import、写锁保护
- 支持 Codex / Claude 宿主集成

## 当前边界

已经稳定可用的部分：

1. 本地知识记录、查询、review、持久化主链路
2. 经验与论文共图检索
3. 多代理环境下的基本写入保护
4. 非破坏式经验演化治理第一版
5. 长文本 capture/论文摘要的初步压缩与分层

仍然属于下一阶段的部分：

1. MCP / query server 这类结构化外部接入
2. 更成熟的研究工作流，如论文分组、方向卡片、筛选视图
3. 更强的 stale 分析，目前仍偏文件 anchor
4. 更系统的图分析，如聚类、热点、时间序列
5. 自动摘要目前是规则式压缩，不是更高级的语义归纳

## 现在最值得做的事

建议优先级如下：

1. **结构化外部接入**
   让外部 agent 或工具能以 API / MCP / query service 形式读取 EKG，而不只靠 CLI 和 Hook。

2. **研究工作流增强**
   在 papers + survey 之上继续做论文筛选、方向归纳、候选池管理。

3. **stale 与失效治理增强**
   从文件级 stale 走向模块级、概念级和关系级 stale 提示。

4. **高级图分析**
   增加聚类、热点、链路强弱、版本扩散等分析能力。

## 最近已完成的重要变化

- paper-aware workflow 与相关文档同步
- 非破坏式 `supersede` 经验演化治理
- `SUPERSEDED` 状态、排序降权、推荐当前版本提示
- panel 中的版本链显示
- task-complete 的 capture skip 原因提示
- 论文与 capture 长文本的紧凑摘要压缩
- 论文 full abstract + compact summary 的分级存储

## 当前测试状态

最近一次本地验证结果：

- `19/19 tests passed`

## 当前结论

如果只保留一句判断：

**EKG 现在已经不是“经验笔记工具”，而是一个以 SQLite 为主存储、支持经验层、论文层、review gate、演化治理和宿主 Hook 的本地知识中间层原型。下一步的关键，不是继续堆命令，而是把外部接入和研究工作流做成熟。**
