## 下一步建议

建议下一步优先做：

1. **自动 ingest 原型**：从 git commit、任务完成记录、对话摘要里生成候选经验。
2. **stale 检测**：当锚定文件大幅变化时把经验标记为 `NEEDS_REVIEW`。
3. **图谱导出**：支持 GraphML/GEXF，方便用 Gephi 等工具分析。
4. **MCP/query server**：让 Codex、Claude 和其他 agent 能结构化调用 EKG。

## 明确待办：Phase 2 缺口

### 1. 自动 ingest

从 commit / task / 对话摘要里自动生成候选经验。

第一版建议只做候选生成：

- 输入源：git commit、任务完成摘要、hook capture 信息。
- 输出：`capture candidate`。
- 默认状态：`NEEDS_REVIEW`。
- 默认置信度：`UNCERTAIN`。
- 不允许直接写入正式经验库。

### 2. stale 检测

文件变了就自动提示相关经验可能过期，并进入 `NEEDS_REVIEW`。

第一版建议按文件级别做：

- 记录 anchor 文件的 hash / mtime / commit。
- 检测文件删除、文件内容大幅变化、项目路径变化。
- 命中后把相关经验标记为 `NEEDS_REVIEW`。
- 后续再升级到函数名、组件名、模块名等语义锚点。

这两个功能是下一阶段最高优先级。
